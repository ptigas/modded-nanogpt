Muon optimizer intuition

Why orthogonal, spectral-norm–aware updates help

These mini applets are toy visualizations. They don’t simulate a full LLM, but they give geometric intuition: Muon re-shapes gradient updates at the matrix level so you don’t just follow a single dominant direction. The Nyström applet at the end shows how you can approximate big “curvature” matrices using only a few columns.

1. Big picture Conceptual walkthrough
A weight matrix in a transformer layer doesn’t just have a “size” – it has directions (its singular vectors) and scales (its singular values).

• With Adam/AdamW, the update can become dominated by a few high-variance directions in the gradient. You’re effectively following a narrow valley dominated by one mode.
• Muon takes the gradient momentum for a matrix and orthogonalizes it (think “SVD, then keep UVᵀ) so the update acts like a balanced rotation + scaling, rather than stretching along just one direction.

The applets below illustrate that:
• Applet A: “many gradients collapse to one direction” vs “spread across orthogonal directions”.
• Applet B: plain gradient descent in an anisotropic quadratic vs a “Muon-like” preconditioned step.
• Applet C (Nyström): how to approximate a big PSD matrix (e.g. a kernel or curvature-like matrix) using only a few of its columns.
2. Many gradient directions vs orthogonal Muon update Applet A
Think of each arrow as a column of a gradient matrix. If they all point in nearly the same direction, your update is effectively one-dimensional. Muon orthogonalizes and rescales them so you use multiple independent directions.

• When the slider is on the left, the green arrows are close to the red ones: several columns of the gradient point in almost the same direction.
• As you move the slider right, you mix in the blue orthogonal directions. This is analogous to Muon’s (MMᵀ)-1/2M step: it spreads the update across multiple directions with comparable magnitude.
• That’s why Muon behaves like a cheap second-order method: it stops a single mode from dominating and lets the model learn richer structure in parameter space.
In the real optimizer, the orthogonalization is done via a Newton–Schulz iteration on the momentum matrix rather than an explicit Gram–Schmidt in 2D, but the geometric idea is the same.
3. Walking down an anisotropic valley Applet B
Now imagine a simple quadratic loss: f(x) = ½ (9x₁² + x₂²). This is a skinny valley: steep in x₁, shallow in x₂. Plain gradient descent “zig-zags” slowly. A curvature-aware update (like Newton or a Muon-style preconditioner) rescales directions so steps are more direct.
Descent paths in a skinny valley
Purple contours: level sets of the loss. You start at the big dot. Red path: plain gradient descent. Green path: “Muon-like” preconditioned descent.
Step size (learning rate)
Smaller Larger η = 0.18
• The red path uses plain gradient descent: x ← x − η ∇f(x). Because the curvature is 9× larger in x₁ than in x₂, the steps overshoot and zig-zag.
• The green path uses a simple “Muon-like” preconditioner that rescales directions based on curvature (here just a diagonal matrix). This roughly equalizes effective curvature, so steps move more directly toward the minimum.

In Muon, something analogous happens at the matrix level: the (MMᵀ)-1/2 factor plays the role of a spectral, second-order-ish preconditioner that equalizes update magnitudes across singular directions of the weight matrix.
This toy example uses a 2D quadratic and a diagonal preconditioner. Real Muon uses the gradient’s matrix structure instead of an explicit Hessian, but the qualitative effect on the optimization path is similar.
4. Nyström column sketching: big matrices from a few columns Applet C
In second-order-ish methods, you often want to use a big PSD matrix: a kernel matrix, a Fisher matrix, or something curvature-like. Forming it exactly is too expensive.

The Nyström trick says: “I don’t need the whole matrix, I just need a few of its columns (landmarks), and I’ll reconstruct an approximation from those.” That’s very similar in spirit to what you’d do if you tried to approximate Muon’s spectral preconditioner with a low-rank sketch.

• We place 20 points along a line, build their RBF kernel matrix K (left heatmap).
• Then we randomly pick m columns (landmarks). Nyström builds K̃ = C W⁻¹ Cᵀ, where C is those columns, and W is the small m×m submatrix on the landmarks. That’s the right heatmap.
• As you increase m, the approximation improves: more columns = a richer low-rank sketch.

Intuition link to Muon / second-order: if you ever wanted a full spectral preconditioner but couldn’t afford the complete matrix, Nyström is the conceptual move: “use a few well-chosen columns to capture most of the structure”.
In practice, Nyström is used with smarter column sampling (e.g. via leverage scores). Here we just sample uniformly to keep the picture simple and visual.
